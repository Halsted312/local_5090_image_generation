## 1. Executive Summary

PromptPics is an AI-powered image generation platform that turns natural-language prompts into high-quality images and includes a unique prank-link system for playful, personalized reveals. The service combines several advanced open-weight image models—such as **FLUX.1-dev**, a 12-billion-parameter rectified flow transformer from Black Forest Labs, **RealVisXL V4.0**, a photorealistic SDXL fine-tune, and **Stable Diffusion 3 Medium**, Stability AI’s latest open text-to-image model—behind a single, easy-to-use interface.

Unlike many generic AI image sites, PromptPics is designed to:

* **Route each prompt to the most appropriate model automatically** (portraits vs. landscapes vs. logos, etc.),
* **Run on a dedicated local NVIDIA RTX 5090 GPU** for fast, private processing, and
* Offer a **“prank link” feature** that lets users share links that look like normal generators but secretly show their own custom images when specific trigger prompts are used.

## 2. Core Product Description

PromptPics is an **AI image generation and prank-link platform** accessible at `promptpics.ai`. Users type a description (“a cozy coffee shop at sunrise in watercolor style”), optionally tweak a few settings, and receive one or more images generated by advanced diffusion-based models. The system interprets text prompts, chooses the best engine for the job through a routing layer, and delivers images in seconds.

From a user’s perspective, PromptPics feels like a **professional, modern image generator**:

* Clean, light SaaS-style landing page.
* Single prompt box with optional advanced settings (steps, guidance, model hint).
* A results grid showing the generated image, the model that was used, and a short explanation of why the router chose it.

Under the hood, PromptPics is a **multi-model orchestrator**: it can call different image generation models depending on the content of the prompt (e.g., photoreal portraits vs. stylized art vs. typography-heavy logo prompts). This is a key differentiator compared with “one-model-only” competitors.

## 3. Multi-Model Engine & Routing

### 3.1 Models Supported

PromptPics’ backend is built around a set of state-of-the-art open models:

1. **FLUX.1-dev (“flux_dev”)** –

   * A 12-billion-parameter rectified flow transformer for text-to-image generation, released by Black Forest Labs.
   * Known for strong prompt adherence, high visual quality, and flexibility across styles.

2. **RealVisXL V4.0 (“realvis_xl”)** –

   * An SDXL-based model fine-tuned specifically for **photorealism**, especially faces and human subjects.
   * Widely regarded by community users as one of the best SDXL variants for realistic humans.

3. **Stable Diffusion 3 Medium (“sd3_medium”)** –

   * A new Multimodal Diffusion Transformer from Stability AI.
   * Designed to produce improved image quality, clearer typography, and better understanding of complex prompts while being efficient enough to run on consumer hardware.

4. **Logo-oriented SDXL Base (“logo_sdxl”)** –

   * A Stable Diffusion XL base model tuned for **logos, icons, simple flat shapes, and text-heavy compositions**.

Each model has well-defined strengths:

* FLUX.1-dev → all-rounder for general scenes, stylized art, concept pieces.
* RealVisXL → photo-real people, portraits, lifestyle imagery.
* SD3 Medium → complex layouts, scenes with multiple objects, text in the image.
* SDXL-Logo → logos, icons, wordmarks, badges.

### 3.2 Router Logic

PromptPics includes a **routing layer** that analyzes each incoming prompt and decides which model to use. The router combines:

* **Heuristic scoring** based on keyword lists (e.g., “portrait”, “photo”, “logo”, “poster”, “dashboard”, “anime”), and
* An optional **small LLM router** (e.g., an 8B-parameter Llama-family model) that reads the prompt and outputs structured JSON indicating:

  * which model it prefers,
  * confidence scores per model,
  * tags (e.g., “portrait”, “logo”, “complex_scene”), and
  * a natural-language reason (“The prompt describes a realistic headshot, so RealVisXL is best.”).

That JSON is stored alongside the generation for later analysis. This multi-model routing pattern is similar to what some enterprise AI platforms use for “LLM routing” or “ensemble models,” but applied to image generation.

From a marketing standpoint, this lets PromptPics claim:

* **Smart model routing** – “Auto picks the best AI engine based on your prompt.”
* **Model transparency** – the UI can show **what was used and why**, building trust.

## 4. Prank Links – Unique Differentiator

One of PromptPics’ most distinctive features is its **prank-link system**.

### 4.1 What a Prank Link Is

A prank link is a **special URL** (e.g., `promptpics.ai/p/a7x9q2lm`) that looks like a normal PromptPics generator to the recipient:

* The page has the same prompt box and “Generate” button.
* Most prompts behave exactly like the standard AI generator.

However, the owner of the prank link can configure **trigger prompts** and **custom images**:

* For example:

  * Trigger 1: “Who is the best dad?” → use uploaded photo of your father.
  * Trigger 2: “Most beautiful girl in the world” → show your partner’s photo.
* When a friend types one of those trigger prompts, PromptPics shows the prank image **instead of** a generated one.

To everyone else, the link behaves like a regular AI image site. It’s essentially a **personalized reveal mechanic**: surprise birthday messages, inside jokes, or fun reveals for family and friends.

### 4.2 How Prank Creation Works (builder view)

The site has a **“Customize / Prank Links”** area where the creator can:

1. Create a new prank set – the backend returns:

   * a **builder slug** (secret, for editing), and
   * a **share slug** (short URL to send to others).
2. Add **multiple prompt–image pairs** to the prank set:

   * type the trigger prompt,
   * upload a custom image,
   * see it appear in a table with thumbnail, timestamp, and match count.
3. Edit or delete triggers:

   * change the text,
   * replace the prank image (keeping versions),
   * remove triggers that are no longer relevant.
4. Share the public URL – which is indistinguishable from a normal generator for untriggered prompts.

### 4.3 Matching Logic

At generation time for `p/:shareSlug`:

* The system reads the user’s prompt.
* Compares it against the prank triggers using:

  * simple exact/substring logic, and/or
  * LLM-based semantic similarity (“Is ‘prettiest woman ever’ close to ‘most beautiful girl in the world’?”).
* If a match is found:

  * The custom prank image is served.
  * The event is logged as a prank match in `generation_logs`.
* If no match:

  * The prompt is passed to the normal router + image generation pipeline.

This design gives PromptPics a **playful, shareable use case** that differentiates it from purely serious AI image tools while still supporting professional use.

## 5. Architecture & Privacy Angle

A key message for Ads and landing pages: **PromptPics runs on a dedicated local GPU**, not a generic cloud backend. The current setup uses an **NVIDIA RTX 5090** plus a high-core-count CPU and large RAM pool to host all models and the router.

This has several marketing and privacy benefits:

* **Speed** – high-end local hardware can generate images quickly without waiting in shared queues.
* **Privacy & control** – content never leaves the machine except to be sent back to the user in the browser, which appeals to privacy-conscious users compared to pure cloud SaaS. Articles about local SD3 Medium, for example, highlight the appeal of running advanced models on consumer hardware without sending data to external servers.
* **Predictable cost** – fixed hardware cost rather than per-image cloud tokens.

From a technical standpoint, the stack is:

* **Frontend**: React + TypeScript SPA (hosted on Replit, served at `promptpics.ai`), using Tailwind/Radix for UI, TanStack Query for API calls.
* **Backend**: FastAPI on the local machine, exposed via **ngrok** as `app.promptpics.ai`.
* **Models**: FLUX.1-dev, RealVisXL, SD3 Medium, and SDXL-Logo via Hugging Face `diffusers` or similar.
* **Database & logs**: Postgres or SQLite for `generation_logs` and `prank_sets`, plus filesystem for full-resolution images and thumbnails.

You can use phrases like “**on-device AI**,” “**local GPU inference**,” or “**no raw prompts sent to third-party clouds**” in copy, which align with growing trends around local generative models.

## 6. User Flows & Use Cases

### 6.1 Quick Generate Flow

1. User lands on `/` and sees a hero section plus a generator card.
2. They type a prompt and click **Generate**.
3. The frontend calls `POST /api/generate` with `engine: "auto"`.
4. Backend:

   * routes the prompt to the best model (router logs why),
   * generates the image on the GPU,
   * returns base64 + thumbnail + routing metadata.
5. Frontend displays the image with badges like:

   * “Used: FLUX.1 Dev”
   * “Router picked: RealVisXL (coming soon)”
     (while other pipelines are still being rolled out).

Use cases:

* Creators prototyping album covers, book covers, posters.
* Marketers testing ad visuals or social media posts.
* Casual users making memes, concept art, or reference images.

### 6.2 Prank Link Flow

1. Creator goes to `/customize/new` or a similar entry point.
2. Creates a prank set, receives:

   * Builder URL (for editing),
   * Share URL `https://promptpics.ai/p/abc12`.
3. Adds multiple triggers with custom images.
4. Sends share URL to friends, family, or social media.
5. Recipients visit the link and:

   * For normal prompts → see AI-generated images (impressive and believable).
   * For special trigger prompts → see the prank image.

Use cases:

* Birthday or anniversary reveals.
* Inside jokes among friends or coworkers.
* Marketing campaigns that reveal a hidden visual when the right phrase is entered.

### 6.3 Dashboard / History

The site can show **recent generations** and **pranks created** for a given session:

* What prompts were used.
* Which model the router chose.
* Which pranks were triggered and how often.

This is useful both for:

* Users tracking their own usage, and
* The owner (you) to see how the router is performing.

## 2. Deep-Dive Feature Descriptions

### 2.1. Smart Multi-Model Routing

Most AI image tools force users to pick a single model and hope it fits every task. PromptPics takes a different approach. It hosts multiple specialized engines:

* An all-purpose model for general art and scenes.
* A portrait-tuned model optimized for human faces and skin tones.
* A typography-aware model better at rendering crisp text and complex layouts.
* A logo-friendly model tuned for flat shapes, icons, and brand marks.

A **routing layer** scores each incoming prompt on several dimensions: whether it contains portrait keywords, logo cues, requests for text-on-image, complexity markers (“and”, “with”, “in the background”), or stylistic hints (“anime”, “watercolor”, “minimalist”). Based on these signals, PromptPics assigns a probability to each engine and chooses the one with the highest score. Where prompts are ambiguous, a lightweight LLM can be consulted to make a final decision, returning JSON with confidence scores and reasoning that can be displayed in the UI or stored for analysis.

For your Ads and messaging, you can frame this as:

* “Automatic model selection for better results.”
* “Less guesswork, more good images.”
* “No need to know which AI model does what—PromptPics routes for you.”

### 2.2. Fast, High-Quality Image Generation

PromptPics is designed to **turn ideas into visuals quickly**. Running on a high-end GPU means it can:

* Generate 1024x1024 or similar resolution images in seconds.
* Use higher step counts or more complex samplers without making users wait.
* Test multiple variations of a concept (e.g., 3–4 takes on the same prompt) within a single session.

The platform returns images in high resolution along with smaller **thumbnails** optimized for quick previews and gallery views. Users see their most recent creations in a timeline, with model tags and timestamps, making it easy to track what they’ve tried and refine prompts over time.

### 2.3. Prank Sets & Trigger Prompts

The prank system is built around the idea of **prank sets**:

* A prank set is a collection of trigger prompts and associated images.
* Each set has:

  * a private **builder link** for editing and management, and
  * a public **share link** for viewers.

Within the builder interface, users can:

* Add new triggers with a simple form: type the phrase, upload the image.
* See a table of all triggers, each with:

  * a thumbnail,
  * text of the trigger prompt,
  * date created,
  * “matches” count (how many times people hit that trigger), and
  * buttons to edit or delete.

The prank matching logic can be tuned from strict (exact text only) to more fuzzy (semantic similarity), so “world’s best dad” can still match “best dad ever” or “my dad is the best in the world.”
